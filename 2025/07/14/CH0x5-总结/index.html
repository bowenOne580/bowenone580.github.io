CH0x5 总结

DP 概念

DP 分为三步：阶段、状态、决策。

阶段，指计算时的顺序性，总是用上一个阶段的结果推导下一个阶段的结果，这样的转移顺序具有无后效性的特点。

状态，指一个阶段中，每个能够转移到的点。

决策，指转移状态的选取，即一个点被哪些点转移，或者一个点能够转移到哪些点。

如果将 DP 的过程看作一张图，那么这张图一定是有向无环图，其中每个节点是一个状态，决策是一个状态到另一个状态的有向边。

动态规划的基础是递推，它能够正确求解问题的关键，在于之前求出每一个状态的值，都是最优的，不会再被之后的状态更新，这叫做最优子结构性质。

DP 的时间复杂度和状态总数有很大的关系，因此，要寻找状态表示所需维度最小的 DP。

线性 DP

指阶段划分是线性的 DP，即从 $i-1$ 推到 $i$。

基本问题：LIS（$O(n\log n)$），LCS（$O(n^2)$），数字三角形（$O(n^2)$）。

SP15637 GNYR04H - Mr Youngs Picture Permutations - 洛谷：寻找最简状态表示。

P10954 LCIS - 洛谷：决策集合具有单调性时，边加入边维护。

P2893 [USACO08FEB] Making the Grade G - 洛谷：推结论。

SP703 SERVICE - Mobile Service - 洛谷：看似有三个维度，实际上上一次移动后，必然有点在终点处，所以只有两个维度。

P1006 [NOIP 2008 提高组] 传纸条 - 洛谷：注意不能跑两遍 DP，这样有可能不是最优解。要维护四个坐标，但实际上固定移动步数后，就只用维护两个坐标了。

P10971 Cookies - 洛谷：先贪心，之后朴素的做法是维护分配的饼干总数和最后一个人拿的数量，但实际上我们只关心相对关系，所以可以默认最后一个人拿 $1$ 块，之后再整体 $+1$，即可遍历到所有状态。

可以发现，线性 DP 主要难点在于确定一个状态的多个维度。有时候多个维度可以被一个维度表示，有时决策时决策集合是只加不减的，这些都可以优化时间复杂度。

区间 DP

区间 DP，不止是区间上的问题，还可以是很多满足相邻性质的问题。它们的特点是，每次合并时两个状态一定是相邻的，比如序列上的相邻，或者让一个区间被一个数代表，每次合并两个相邻的数。

阶段就是区间的长度，一般先处理出 $1$ 的值，之后从 $2$ 开始，依次增加长度。枚举区间的左右端点，分别从较小的区间转移过来。

P1775 石子合并（弱化版） - 洛谷：模板题。

P4342 [IOI 1998] Polygon - 洛谷：断环为链，之后每次删除一条边，相当于合并两个相邻的点。

P10956 金字塔 - 洛谷：枚举子树的区间，由于每个子树所代表的都是区间，且一个根节点下的子树的区间是相邻的，所以使用区间 DP。

树形 DP

一般以子树为状态，枚举一个点的所有儿子，从儿子的子树将信息转移到父节点，注意这里的转移只能访问儿子，而不能访问更深的节点，否则复杂度就不对了。树形 DP 要满足能从儿子的状态推到父亲的状态。

P1352 没有上司的舞会 - 洛谷：将每个点是否选作为两种状态。

P2014 [CTSC1997] 选课 - 洛谷：树上背包，这里是分组背包。树上背包的时间复杂度是 $O(n^2)$。

P10974 Accumulation Degree - 洛谷：换根 DP，重点在从全局信息和子树信息，推出除子树外的其它信息。

树状数组优化 DP

P12930 [USACO4.3] 逢低吸纳 Buy Low, Buy Lower 加强版 - 洛谷

最长下降子序列，求本质不同的方案数。设 $f_i$ 表示以 $i$ 结尾的序列长度，每次转移时实际上是找大于 $a_i$ 的所有数的 $f$ 的最大值。

还要维护方案数，朴素的维护方式是，求所有大于 $a_i$ 的数中和最大值相等的位置的方案之和。考虑到只需要维护最大值相关的那些数，所以可以在维护树状数组的时候顺便合并。

这也启示我们，树状数组可以向线段树一样维护多个信息，只要两个节点可以正确合并。
