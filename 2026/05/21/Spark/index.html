简介

Spark Core 是用于取代 MapReduce 的功能的，做批处理计算，将数据放到内存中，所以很快（MapReduce 是每次都从硬盘读取的）。

在上面有许多框架，支持其它形式的计算，相当于一个接口，最终还是将计算任务交给 Spark Core 进行处理。

但我们平时编写计算任务，可以直接和顶层的模块打交道，这样能让体验更接近与传统的单机上进行计算的体验。

RDD

RDD（弹性分布式数据集） 是 Spark 中进行计算的基本模块，所有 RDD 都是只读的，这也就意味着一切对 RDD 的操作实际上都是在副本上进行的，最后返回这个副本，这也是代码中需要定义 4 个 RDD 的原因。

弹性体现在，如果某个 RDD 中的某个部分计算出错或结果丢失，不需要再从 HDFS 开始重新计算一遍，只需要从上一层 RDD 进行一层计算即可恢复，所需的计算量大大减少。

RDD 操作可以分为转换的动作，RDD 采用懒计算策略，只有在真正执行动作时才会开始计算，否则只会记录转换信息，保证能完成计算。

容易发现，宽依赖恢复所需的计算量比窄依赖大很多，所以要尽量少的在代码中造成宽依赖。

会造成宽依赖的算子：

重分区（repartition）类算子，如： repartition 和 coalesce；ByKey 类算子(除了计数类的，如 countByKey) 如：groupByKey 和 reduceByKey；以及Join类算子，如：cogroup 和 join

    再划分阶段，每出现一次宽依赖，就划分一个阶段，这样能保证每个阶段内部，所有节点都是一对一的关系，形成若干条链，这样就可以将每个链视作一个任务，划分 Task，分配给分布式集群计算了。
